# Legacy-System-Migration-Gap-Analysis-OAT-Case-Study
🧾 Project Overview

This project demonstrates my role in analyzing and validating a cloud-based system developed to replace a legacy IBM AS/400 system.

The organization initiated a digital transformation by replacing its legacy system with a modern web-based solution. However, during the User Acceptance Testing (UAT) phase, significant mismatches were identified between user expectations and system functionality.

🎯 Objective
Identify gaps between current business processes and the new system
Validate system functionality against real user requirements
Support UAT and Operational Acceptance Testing (OAT)
Improve requirement clarity through structured analysis
👩‍💻 My Role

System Analyst (Transitioned from Developer Role)

Collaborated with cross-functional departments
Conducted requirement gathering sessions
Analyzed existing AS/400-based workflows
Matched “as-is” processes with the “to-be” system
Identified functional gaps and mismatches
Designed test cases for system validation
Contributed to persona creation for user behavior analysis
🔍 Key Activities
1. Business Process Identification
Conducted meetings with departmental users
Documented current workflows (“As-Is” processes)
Identified inefficiencies and manual workarounds
2. Gap Analysis
Compared user requirements with the developed system
Identified missing functionalities and mismatches
Documented gaps for vendor feedback
3. User Persona Development
Created personas representing key user roles
Identified primary tasks and system expectations
Used personas to guide testing and validation
4. UAT Support & Test Case Design
Participated in module-level system testing
Designed detailed test cases covering:
Functional scenarios
Edge cases
Error handling
Logged and tracked system defects
